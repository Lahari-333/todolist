System Architecture

Tenant Isolation

Every record includes an org_id. All queries filter by org, so organizations never see each other's data.

RBAC

Admins can edit/delete any task in their org. Members can only manage tasks they created.

Authentication

In production: JWT with org_id and role in payload. OAuth via Google/Facebook/GitHub would require a backend server.

Containerization

Production deployment would use a Dockerfile + docker-compose.yaml orchestrating app + PostgreSQL containers. Not possible in browser-only environment.

Data Schema

tasks: { org_id, title, description, status, priority, created_by, created_by_role, created_at, updated_at }
audit: { org_id, action, target_task, user_name, timestamp }
